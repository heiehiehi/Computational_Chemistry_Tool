# energy.py: 全自动高通量结合自由能计算工具

## 📖 工具简介
在计算化学和 AI 辅助药物设计（AIDD）中，使用 `gmx_MMPBSA` 评估批量小分子（Hits）的结合自由能是一项耗时且易错的工作。特别是当处理复杂靶点（如带有磷酸化修饰的激酶 JAK2）时，不同力场间的拓扑转换经常会导致程序崩溃。

本脚本基于 Python 驱动 WSL (Windows Subsystem for Linux)，专为**多靶点、多配体的批量 MM/PBSA 计算**设计。它集成了 MPI 多核加速、断点续传保护，并内置了针对 AMBER/GROMACS 拓扑转换错误的智能修复逻辑。

## ✨ 核心特性
* **🚀 极速多核并行**：自动调用 `mpirun --oversubscribe -np 8`（可根据宿主机核心数调整），压榨 CPU 性能，显著缩短数百帧轨迹的能量评价时间。
* **🛡️ 蓝屏/断电级容灾 (断点续传)**：基于输出文件验证逻辑。若计算中途因 CPU 过热蓝屏或断电，再次运行脚本时会自动跳过已生成的 `FINAL_RESULTS_MMPBSA.dat` 体系，无缝继续下一个任务。
* **🔧 智能拓扑修复 (Topology Auto-Fix)**：
    * **痛点**：当 GROMACS 拓扑（尤其由 Sobtop 等生成）包含特殊修饰残基（如磷酸化酪氨酸 PTR）时，`gmx_MMPBSA` 底层的 `ParmEd` 翻译器常会报出 `Not all improper torsion parameters found` 的致命错误。
    * **解决**：脚本内置靶点识别机制。当检测到特定靶点（如 JAK2）时，自动利用 `sed` 正则表达式对 `topol.top` 进行无损微创手术，强制补全缺失的二面角参数（`180.00 4.60240 2`），彻底根治崩溃顽疾。
* **🤫 纯静默挂机模式**：强制引入 `-nogui` 参数，屏蔽数据分析后处理的自动弹窗，真正实现“一键启动，睡醒收图”。

## 📂 依赖的目录结构
脚本默认你的 MD 结果文件按照以下标准结构组织（路径可在代码的 `BASE_DIR` 中修改）：

```text
E:\GROMACS\Two Target\
├── energy\
│   └── mmpbsa.in               # MMPBSA 全局参数控制文件 (需包含正确力场)
├── STAT3\                      # 靶点 1 目录
│   ├── Hit1\                   # 化合物 1 的 MD 目录
│   │   ├── md_fit.xtc          # 去除 PBC 并叠合好的轨迹
│   │   ├── md.tpr              # MD 运行输入文件
│   │   ├── topol.top           # 拓扑文件
│   │   └── index.ndx           # 索引文件 (需包含 Receptor=1, Ligand=13 组别)
│   └── Hit2\
├── JAK2\                       # 靶点 2 目录
│   └── Hit-RUX\
└── MMPBSACal\                  # 自动创建的输出总目录，按靶点和 Hit 存放结果
```

## ⚙️ 环境要求
* **系统平台**：Windows 10/11 + WSL2 (Ubuntu)
* **环境依赖**：
    * 已安装并配置好 Conda（环境中需包含 `gmx_MMPBSA`）
    * OpenMPI (`sudo apt install openmpi-bin`)
    * GROMACS 5.x 及以上版本

## 🚀 快速开始
1. 确保 WSL 正在运行，且 Conda 环境名与脚本内的 `conda activate gmxMMPBSA` 匹配。
2. 确保所有需要计算的 Hit 目录中已经包含了去重叠后的 `.xtc`、`.tpr`、`.top` 和 `.ndx`。
3. 在 Windows 环境下直接运行 Python 脚本：
   ```bash
   python auto_mmpbsa.py
   ```
4. 观察终端输出的进度条，所有的计算结果将被自动归档至 `MMPBSACal` 文件夹中。

# ProLIFget.py: MD 轨迹蛋白-配体相互作用指纹高通量分析工具

## 📖 工具简介
在计算机辅助药物设计（CADD）中，分子动力学（MD）模拟是验证小分子抑制剂结合稳定性的金标准。本工具基于 `MDAnalysis` 和 `ProLIF` 库构建，专为**高通量、多体系（Hits）的 MD 轨迹分析**设计。

它可以全自动遍历靶点目录下的所有分子轨迹，精准捕获蛋白与配体在模拟过程中的动态非共价相互作用（氢键、盐桥、π-π 堆叠、疏水作用等），并一键输出达到 SCI 发表级别的可视化图表与 Origin 数据源。

## ✨ 核心特性
* **📂 全自动批处理**：只需指定靶点主目录（如 `STAT3` 或 `JAK2`），脚本会自动扫描所有 `Hit*` 文件夹，逐一处理，无需人工干预。
* **🧬 智能残基序号重映射 (核心亮点)**：GROMACS 在生成 `.tpr` 拓扑时会重排原子和残基序号，导致输出的残基编号与真实晶体结构不符。本脚本通过对齐 `frame*.pdb` 与 `md.tpr`，自动将内部序号完美映射回晶体学真实编号，确保图表标注绝对准确。
* **🧹 数据智能提纯**：自动计算各类相互作用在轨迹中的发生频率（Occupancy），智能过滤总占据率 `< 15%` 的背景噪音，并筛选出贡献最大的 Top 10 关键氨基酸位点。
* **🎨 SCI 级图表双轨输出**：
    * **直出图**：自动绘制高颜值的相互作用堆叠柱状图（`.png`, 300 DPI），采用标准化配色字典，所见即所得。
    * **数据源**：同步导出汇总清洗后的矩阵数据（`*_Occupancy_Origin.csv`），方便随时拖入 OriginLab 进行二次个性化作图。
* **🛡️ 内存安全机制**：内置绘图内存释放逻辑（`plt.close`），即使连续处理数十个几百纳秒的轨迹，也能保持极低的内存占用，杜绝批处理崩溃。

## 📂 目录结构与依赖文件
程序会自动在当前运行目录下创建同名靶点文件夹存放结果。你需要确保原始输入目录满足以下基础结构：

```text
E:\GROMACS\Two Target\STAT3\      # 在 BASE_DIR 中指定的靶点目录
├── Hit1\                         # 待处理的复合物体系 1
│   ├── md.tpr                    # MD 运行输入拓扑文件
│   ├── md_center.xtc             # 已去除 PBC 周期性边界并居中的轨迹文件
│   └── frame_0.pdb               # 任意一帧的 PDB 文件（用于提供真实的残基序号）
├── Hit2\
│   ├── md.tpr
│   ├── md_center.xtc
│   └── frame_0.pdb
...
```

**产出结果结构：**
```text
当前运行目录\
└── STAT3\                        # 自动提取靶点名称作为总目录
    ├── Hit1\
    │   ├── Hit1_Occupancy_Origin.csv    # 可导入 Origin 的干净数据表
    │   └── Hit1_Occupancy_Stacked.png   # 相互作用频率堆叠柱状图
    ├── Hit2\
    ...
```

## ⚙️ 环境依赖
建议使用 Conda 创建独立环境运行：
```bash
# 核心依赖库
conda install -c conda-forge mdanalysis prolif pandas matplotlib
```

## 🚀 使用指南
1. 打开脚本，定位到文件顶部的 `🌟 全局设置区`。
2. 将 `BASE_DIR` 修改为你实际存放 MD 结果的本地靶点目录路径（Windows 下请保留 `r` 前缀防转义）。
3. 在终端或 IDE 中运行脚本：
   ```bash
   python auto_prolif.py
   ```
4. 程序会打印清晰的带前缀日志（如 `[Hit1] ✅ 处理成功！`），执行完毕后直接去对应文件夹收获图表即可。

## 💡 进阶修改建议
* **相互作用阈值**：如果想展示更多的弱相互作用，可修改代码中的 `> 0.15`（即发生频率 15%）；若只想看强相互作用，可调高至 `0.3`。
* **配色字典**：若对生成的柱状图配色不满意，可直接修改代码中的 `interaction_colors_dict` 十六进制色值。
